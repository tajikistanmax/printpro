# ПОЛНЫЙ ТЕКСТ ТЗ

## 1. Назначение системы

PrintPro — единая программа для автоматизации работы типографии, копицентра, рекламного
производства и полиграфического бизнеса. Объединяет: POS-кассу, приём заказов, склад
материалов, производство, дизайн-макеты, клиентов, оплату и финансы, сотрудников, отчёты,
онлайн/офлайн работу, синхронизацию между филиалами.

Цель — вся работа типографии в одной системе: от приёма клиента до печати, оплаты,
списания материалов и выдачи готового заказа.

## 2. Основные модули

### 2.1. Авторизация и роли
Роли: Администратор, Кассир, Менеджер заказов, Дизайнер, Печатник/оператор, Складовщик,
Бухгалтер, Руководитель, Суперадминистратор.
Права для каждой роли: просмотр, создание, редактирование, удаление, подтверждение, печать,
возврат, скидка, доступ к кассе, доступ к складу, доступ к финотчётам.

### 2.2. Главный дашборд
Показывает: заказов за сегодня, сумму продаж за день, активные/срочные заказы, в производстве,
завершённые, остатки важных материалов, задолженности клиентов, прибыль, расходы, уведомления.
Быстрые кнопки: Новый заказ, Быстрая продажа, Открыть кассу, Склад, Клиенты, Отчёты.

### 2.3. POS-касса
Поиск услуги/товара, добавление в чек, количество, ручная скидка, скидка по клиенту,
оплата наличными/картой/QR/смешанная, возврат, отмена чека, печать чека, открытие/закрытие
смены, X-отчёт, Z-отчёт, история продаж. Касса — отдельное быстрое рабочее место кассира.

### 2.4. Заказы
Поля: клиент, телефон, тип услуги, описание, файл макета, формат, количество, материал,
цветность, срочность, срок, ответственный менеджер, дизайнер, оператор, цена, предоплата,
остаток, статус.
Статусы: Новый, Ожидает макет, В дизайне, Макет на согласовании, Согласован, В производстве,
Готов, Выдан, Отменён, Возврат/переделка.

### 2.5. Производство
Для заказа: номер, клиент, услуга, срок, приоритет, статус, исполнитель, макет, комментарии,
список материалов, история. Сотрудник может: принять в работу, поставить на паузу, отметить
выполненным, отправить на переделку, добавить комментарий, загрузить фото результата, указать
расход материала.

### 2.6. Дизайн-макеты
Список макетов, прикрепление файлов, версии, комментарии клиента, согласование, статус, история.
Статусы: Нужно создать, В работе, Отправлен клиенту, Требует правки, Согласован, Отклонён.

### 2.7. Услуги и прайс-лист
Справочник услуг (печать A4/A3, цветная/ч-б, широкоформат, баннер, визитки, буклеты, наклейки,
ламинация, переплёт, сканирование, дизайн, плоттерная резка, DTF/UV/сувенирка).
Для услуги: название, категория, ед. изм., базовая цена, себестоимость, материал по умолчанию,
время выполнения, активна/неактивна.

### 2.8. Склад и материалы
Материалы: бумага, баннер, плёнка, чернила, тонер, картриджи, ламинат, расходники, готовые
товары, упаковка. Функции: приход, расход, списание, перемещение, инвентаризация, остатки,
мин. остаток, уведомление о нехватке, история движения. Списание при заказе авто/вручную.

### 2.9. Клиенты
Типы: физлицо, компания, постоянный, VIP. Карточка: ФИО/название, телефон, email, адрес, ИНН,
скидка, история заказов, задолженность, комментарии, файлы.

### 2.10. Финансы
Показывает: продажи, оплаты, предоплаты, долги, возвраты, расходы, прибыль, кассовые смены,
зарплаты, закупки. Отчёты: дневной/недельный/месячный, продажи по кассиру, по услугам,
прибыль по заказам, задолженность клиентов, движение денег.

### 2.11. Сотрудники
Список, должность, роль, телефон, логин, статус, история действий, выполненные заказы,
показатели эффективности (заказы, продажи, ошибки/переделки, сумма, активность).

### 2.12. Отчёты и аналитика
Продажи за период, популярные/прибыльные услуги, расход материалов, остатки, эффективность
сотрудников, заказы по статусам, просроченные, задолженности, кассовые операции, общая прибыль.
Экспорт: Excel, PDF, CSV.

### 2.13. Уведомления
События: новый заказ, просрочка, макет согласован, заказ готов, низкий остаток, долг клиента,
закрытие смены, ошибка синхронизации. Каналы: внутри системы, Telegram, SMS, email.

## 3. Онлайн и офлайн режим
Работа без интернета локально (касса, заказы, склад, клиенты, продажи), синхронизация с
центральным сервером при появлении сети. Учёт конфликтов, повторных операций, уникальных
номеров заказов/чеков, истории изменений, защита от потери данных.

## 4. Архитектура
Backend: Node.js + TypeScript + NestJS. Frontend: React + TypeScript. Desktop: Electron.
Mobile: Flutter/React Native. БД: PostgreSQL (центр) + SQLite (офлайн). API: REST + WebSocket.

## 5. Установка
Установщик для Windows/macOS/Linux. Локально автоматически: SQLite, настройки кассы/принтера,
локальный сервер синхронизации. Без ручной настройки БД.

## 6. Нумерация документов
Авто-номера: заказы, чеки, кассовые смены, складские операции, акты списания, приходные.
Пример: ORD-2025-000001, POS-2025-000001, WH-2025-000001.

## 7. Безопасность
Логин/пароль, JWT/refresh, 2FA при необходимости, разграничение прав, журнал действий,
бэкап, шифрование, защита от удаления критических данных. Каждое действие: кто/что/когда/
с какого устройства/какие данные изменились.

## 8. Резервное копирование
Авто/ручной бэкап, восстановление, хранение локально и на сервере.

## 9. Страницы интерфейса
Вход, Дашборд, POS-касса, Заказы, Новый заказ, Производство, Дизайн-макеты, Услуги, Склад,
Приход, Списание, Клиенты, Финансы, Кассовые смены, Отчёты, Сотрудники, Роли и права,
Настройки, Уведомления, Журнал действий.

## 10. Требования к дизайну
Современный, быстрый, понятный, адаптивный, удобный кассиру/админу, крупные кнопки, быстрый
поиск, тёмная и светлая темы. Экраны: мин. 1280×720, комфорт 1440×900, большой 1920×1080.

## 11. Бизнес-процесс заказа
Клиент → менеджер создаёт заказ → макет/дизайнер → согласование клиентом → производство →
списание материалов → оплата → выдача → отчёты.

## 12. Бизнес-процесс кассы
Открыть смену → добавить позиции → оплата → чек → деньги в кассе → закрытие смены → отчёт.

## 13. Бизнес-процесс склада
Приход материала → остаток → списание при заказе → уведомление при низком остатке → отчёт.

## 14. MVP (первая версия)
Авторизация, роли, дашборд, POS-касса, заказы, клиенты, услуги, склад, финансы, отчёты,
настройки, журнал действий.

## 15. Вторая версия
Производство, дизайн-макеты, Telegram-уведомления, офлайн-режим, синхронизация, мобильное
приложение, филиалы, расширенная аналитика, онлайн-оплата.

## 16. Производительность
Быстрый запуск; касса ≤2с, заказ ≤3с, поиск клиента ≤1с, поиск услуги ≤1с; без зависаний.

## 17. Итоговая цель
Единая система: руководитель видит весь бизнес, кассир быстро продаёт, менеджер контролирует
заказы, дизайнер работает с макетами, производство выполняет задачи, склад списывает материалы,
финансы показывают реальную прибыль.
