## Что это за проект
Платформа для **типографии/полиграфии** (не продуктовый магазин — товар по весу не нужен).
Моно-репозиторий:
- `printpro-api` — бэкенд NestJS + Prisma + PostgreSQL
- `printpro-web` — фронтенд Next.js

Продаётся в двух моделях: облако (подписка) и коробка (установка на ПК).

## Как запустить локально
1. `printpro-api/.env` (не в git): `DATABASE_URL="postgresql://postgres:postgres@localhost:5432/printpro?schema=public"`, `JWT_SECRET="..."`, `PORT=3000`
2. `cd printpro-api && npx prisma migrate deploy && npx prisma generate && npx ts-node prisma/seed.ts`
3. API: `npm run start:dev` → http://localhost:3000/api
4. Веб: `cd printpro-web && npm run dev` → http://localhost:3001
5. Вход: `admin` / `admin123`. Касса — вход по PIN (задаётся в «Сотрудниках»).

## 🐛 QA-сессия 2026-07-01 (деньги/приёмка/долги) — исправлено
Полный прогон приёмки и оплат на живой БД. Исправлены и проверены баги учёта:
1. **Оплаты поставщикам не отражались в кассе.** `paySupplierDebt` и оплата при приёмке (`createReceipt`) создавали `CashMovement` без `shiftId` (или не создавали вовсе) → расход не попадал в Z-отчёт, завышал наличные. Теперь расход привязан к открытой смене кассира.
2. **Возвраты (`refund`, `createReturn`) мимо кассы** — та же причина, привязаны к смене.
3. **Переплата заказа** не блокировалась (можно было внести больше суммы) → фантомная выручка и бонусы. Добавлена проверка «оплата ≤ остатку».
4. **Продажа «в долг» помечалась UNPAID**, а не `DEBT`. Теперь `quickSale` с `method=DEBT` ставит `paymentStatus=DEBT`.
5. **Строка «в долг» в Z-отчёте была всегда 0.** Долговая продажа теперь создаёт запись `Payment(method=DEBT)` (сумма=итог, привязана к смене), не меняя `paid/balanceDue`. Строка «в долг» в кассе и отчёте выручки заполняется; реальные деньги (`collected`, `expectedCash`) её исключают — не задваивается. Проверено: долг +250, выручка/наличные без изменений.

**UX-доработки этой сессии:**
- Новая страница **«Долги»** (`/debts`, меню → Продажи): долги клиентов (нам должны) + поставщиков (мы должны) в одном месте, оплата модалкой прямо оттуда.
- **«Сотрудники»**: «Новый сотрудник» — теперь кнопка + модальное окно (было открытой формой).
- **«Закупки»**: оплата долга поставщику — модальное окно вместо `window.prompt`.
- **«Склад»→«Приём»**: заметная плашка-ссылка на «Закупки» (несколько позиций + поставщик + оплата/долг).

**Сроки долгов + приёмка (доработано):**
- **Миграция** `order_debt_due_date`: у заказа поле `debtDueDate`. POS «в долг» принимает срок; на `/debts` — колонка «Срок», подсветка просрочки, кнопка задать срок (`PATCH /orders/:id/debt-due`). Напоминание «Просроченные долги» в колокольчике (ссылка на `/debts`).
- **Приёмка сведена к одной:** простая приёмка со «Склада» убрана (вкладка стала «Перемещение», видна при >1 филиала); вся приёмка — только в «Закупках» (поставщик/цена/оплата/долг). Быстрое действие «Поступление» в карточке товара ведёт в «Закупки».

**Осталось по UX:** у долгов пока нет истории частичных погашений (только текущий остаток).
