## Что это за проект
Платформа для **типографии/полиграфии** (не продуктовый магазин — товар по весу не нужен).
Моно-репозиторий:
- `printpro-api` — бэкенд NestJS + Prisma + PostgreSQL
- `printpro-web` — фронтенд Next.js

Продаётся в двух моделях: облако (подписка) и коробка (установка на ПК).

## Как запустить локально
1. `printpro-api/.env` (не в git): `DATABASE_URL="postgresql://postgres:postgres@localhost:5432/printpro?schema=public"`, `JWT_SECRET="..."`, `PORT=3000`
2. `cd printpro-api && npx prisma migrate deploy && npx prisma generate && npx ts-node prisma/seed.ts`
3. API: `npm run start:dev` → http://localhost:3000/api
4. Веб: `cd printpro-web && npm run dev` → http://localhost:3001
5. Вход: `admin` / `admin123`. Касса — вход по PIN (задаётся в «Сотрудниках»).

## ✅ Сделано (готово к работе)
- **Склад** доведён до продакшена: авторизация на products/stock, валидация DTO, транзакции, защита от перерасхода, мягко удалённые исключены из остатков.
- **Товары:** «Новый товар» — модальное окно, чистая форма (Название, Категория, Ед.изм, Продажная цена, Штрихкод+генерация, Фото + «Дополнительно»: мин.остаток, артикул). Форма редактирования — в том же виде. Закупочная цена — при приходе (не на карточке).
- **Генерация штрихкода** — внутренний EAN-13, уникальный (`GET /api/products/generate-barcode`).
- **Настройки → «Справочники»** — категории и единицы измерения.
- **Ценники** `/price-labels` — печать ценников со штрихкодом (jsbarcode), шаблоны размеров, печать листом.
- **PIN-кассир** — вход на кассе по PIN (токен 12ч, `POST /api/auth/pos-login`), управление PIN в «Сотрудниках».
- **Инфраструктура:** восстановлен модуль `UploadsModule`; `.gitignore` не прячет `src/uploads`.
- **Оборудование кассы:** сканер ШК (keyboard-wedge), дисплей покупателя VFD (Web Serial), **термопечать чеков ESC/POS** (Web Serial, CP866/латиница, авто-печать) — настройки в «Дисплей покупателя».
- **Касса+:** QR-чек (публичная страница `/r/:id`), профессиональный чек с реквизитами, алиасы штрихкодов (товар = неск. ШК).
- **Склад+:** импорт товаров из CSV/Excel, инвентаризация одним листом по филиалу.
- **Синхронизация:** раздел в настройках + кнопка «Синхронизировать сейчас» (server-side цикл), онлайн-фронт сам ходит в облачный API.

## ⏳ Что дальше (по приоритету)
1. **Деплой на Render** — проверить env (`DATABASE_URL`, `JWT_SECRET`) и прогон `prisma migrate deploy` при деплое (см. `docs/02-deploy-render.md`).
2. **Оборудование — остальное** (#6): термопринтер ESC/POS ✅, сканер ✅, VFD ✅. Осталось: **весы** (serialport), **резак/плоттер**, WebUSB-fallback для USB-принтеров без COM.
3. **Мультикасса / физические кассы** (#2): модель `Register` (device_id/api_key), локальная синхронизация касса↔сервер. Нужна, когда у клиента >1 кассы в магазине.
4. **Лицензирование Ed25519** (#3): для коробочной версии (привязка к MAC, `license.lic`, gate). Для облака не нужно.
5. **PIN «только локальная сеть»** — опция вкл/выкл проверки приватного IP (отложено, ломало бы облако).

## ⚠️ Важные решения/ловушки
- Это типография — **не** предлагать «магазинные» функции (весовой товар и т.п.). При неоднозначности — **сначала уточнять**.
- `.gitignore` содержит `**/uploads` — следит, чтобы не пряталось `src/uploads` (есть исключения).
- Владелец коммитит с разных машин — **всегда `git fetch`/`pull` перед работой**.
- Локальная разработка на Windows: перед `prisma generate`/`migrate` останавливать dev-сервер API (держит DLL движка Prisma).
