# Анализ «PrintPro Enterprise ERP» ТЗ — что берём, что отклоняем

Документ фиксирует решение по второму (enterprise) ТЗ: что уже есть, что дорабатываем,
а что сознательно **не делаем**, чтобы не засорять платформу функциями не нашего профиля.

## Сделано (этот этап — производственное ядро)
- **Спецификация материалов услуги** (норма расхода на единицу) + **авто-списание со склада**
  при завершении производственного задания (идемпотентно, флаг `materialsWrittenOff`). [2.1, 2.8]
- **Технологическая карта заказа** — печатная (`/order-card`): характеристики, позиции,
  расчётный расход материалов, поля подписи цеха. [2.1]
- **Загрузка оборудования** — отчёт: очередь / в работе / готово / брак по станкам. [2.2, 2.10]
- **Брак / переделка** — статус `REWORK` с причиной, возврат в работу. [2.2]

## Сделано ранее (закрывает пункты enterprise-ТЗ)
- Себестоимость, прибыль по заказам, выбор оборудования, время услуги [2.1]
- Очередь/этапы производства, сроки, срочность [2.2]
- Онлайн-заказ, загрузка макетов, согласование [2.3 частично]
- Клиенты, задачи, история [2.6 частично]
- POS: кассы/филиалы/офлайн/возвраты/чек/скидки [2.7 частично]
- Склад: бумага/материалы/товары, остатки, авто-списание продажи, закупки, инвентаризация [2.8]
- Дебиторка, движение денег, зарплата [2.9 частично]
- Дашборд, прибыль, выручка, эффективность [2.10 частично]
- REST API [2.11], Telegram + внутренние уведомления [2.12]
- Офлайн + синхронизация точек (свой подход: локальная БД + sync) [2.7]

## Сделано (клиентский/продажный блок)
- **Повторный заказ в 1 клик** — `POST /orders/:id/reorder` (копия позиций и характеристик),
  кнопка «↻ Повторить» в заказе. [2.3, 2.6]
- **Коммерческие предложения (КП)** — модуль `/quotes`: создание (позиции/клиент/срок),
  статусы DRAFT→SENT→ACCEPTED/REJECTED, **КП → заказ** одним кликом
  (`POST /quotes/:id/convert`), нумерация `КП-<узел>-год-NNNNN`. [2.6]
- **Личный кабинет клиента** на публичном сайте (`/cabinet`): вход по телефону (без пароля),
  история заказов со статусами, повтор заказа (`/public/my-orders`, `/public/reorder`
  с проверкой телефона). [2.3]

## Сделано (лёгкие добавки)
- **Расходы кассы по категориям** — у движения кассы (OUT) поле `category`; отчёт
  «Расходы по категориям» за период (`GET /reports/expenses`). [2.9, 2.10]
- **OpenAPI / Swagger** — документация и песочница API на `/api/docs` (+ `/api/docs-json`). [2.11]
- **PWA** — манифест (`/manifest.webmanifest`), иконки, service worker: панель
  устанавливается как приложение, базовый офлайн-фолбэк. [2.13]

- **QR-оплата (как способ)** — `QR` добавлен в способы оплаты кассы (касса/POS/заказы),
  учитывается в Z-отчёте и сводке. Реальный приём через банковский QR-шлюз — отдельно. [2.7]

## Запланировано добавить (нужны ваши данные)
- **Email-уведомления** — требуется SMTP (хост/порт/логин/пароль почты). [2.12]
- **QR-шлюз банка** — реальный приём оплаты по QR: нужен провайдер/банк (Таджикистан). [2.7]

## Сознательно НЕ делаем (засорение / не наш профиль)
- **Онлайн-конструктор продукции** (визитки/футболки/кружки, Canva-подобный редактор) [2.4] —
  огромный модуль; при нужде — внешняя интеграция (Canva), а не свой редактор.
- **Workflow-конструктор маршрутов** [2.5] — enterprise-перебор; этапы производства уже есть.
- **Бюджетирование, прогнозирование, ABC/XYZ** [2.9, 2.10] — тяжёлая BI, низкая отдача сейчас.
- **Нативные мобильные приложения** [2.13] — вместо них адаптивный веб + PWA.
- **Договоры, лиды, кредиторка** [2.6, 2.9] — добавим, если вырастет B2B.
- **Железо-интеграции** (денежный ящик, дисплей покупателя, банк-терминал, авто-проверка
  файлов preflight) [2.7, 2.3] — добавляются под конкретное устройство/потребность.
- **Интеграции с 1С / доставкой / платёжными шлюзами** [2.11] — по конкретному запросу.
