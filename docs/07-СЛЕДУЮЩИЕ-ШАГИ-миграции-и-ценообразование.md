## J. Налог (НДС) — миграция + расчёт

Владелец выбрал «добавить налоговую строку». Ставка уже настраивается в UI
(Настройки → Касса и налоги → `taxPercent`, валидация 0..100, публичный ключ).

### 1. Схема (`printpro-api/prisma/schema.prisma`, модель `Order`)
```prisma
  taxPercent     Decimal       @default(0) @db.Decimal(5, 2)  // ставка налога на момент заказа
  taxAmount      Decimal       @default(0) @db.Decimal(12, 2) // сумма налога (для чека/отчётов)
```
(по желанию — снимок на позицию: `OrderItem.taxAmount`, если нужен налог построчно)

### 2. Миграция
```bash
# остановить dev-сервер API!
cd printpro-api
npx prisma migrate dev --name add_order_tax
```

### 3. Расчёт (`orders.service.ts` — create и quickSale)
- После вычисления `total` (сумма позиций − скидки) прочитать `taxPercent` из настроек.
- `taxAmount = round(total * taxPercent / 100, 2)`; итог к оплате `= total + taxAmount`
  (если цены «с включённым налогом» — наоборот, выделять налог из total; уточнить у владельца
  модель: налог сверху или включён).
- Сохранять `taxPercent`, `taxAmount` в заказе; `balanceDue`/`paid` считать от итога с налогом.

### 4. Документы/отчёты
- Чек/квитанция/счёт: строка «Налог (НДС) N% … сумма».
- `reports.summary`: показать `tax` отдельно; `net`/`collected` — по договорённости.

## K. FK `companyId → Company` (~19 моделей)

**Проверка сирот выполнена — 0 во всех таблицах** (StockMovement, SupplierPayment,
StockReceipt, PurchaseRequest, HeldSale, Return, DocumentCounter, WriteOff, CashShift,
Payment, CashMovement, ClientDebt, ProductionJob, DesignProof, WorkTimeRecord,
PayrollPeriod, SalaryAdvance, SalaryRecord, AuditLog). **FK можно добавлять безопасно.**

### Схема: у каждой из моделей добавить relation (пример для `Payment`)
```prisma
  company   Company @relation(fields: [companyId], references: [id])
```
и в модели `Company` — обратные списки (`payments Payment[]`, `stockMovements StockMovement[]`, …).
`AuditLog.companyId` — nullable, поэтому relation необязательный (`Company?`).

### Миграция
```bash
# остановить dev-сервер API!
cd printpro-api
npx prisma migrate dev --name fk_company_all
```
Prisma создаст `ALTER TABLE … ADD CONSTRAINT … FOREIGN KEY (companyId) REFERENCES "Company"(id)`.
Сирот нет — миграция пройдёт без ошибок.

## I. Активация серверного ценообразования (движок уже готов)

Готово: `PricingService` + `POST /pricing/preview` (тираж/размер/площадь/опции/дизайн/минтираж).
Осталось подключить к формам и (опц.) закрепить в чек-ауте — это меняет UX формы заказа,
поэтому проверять в браузере.

### Формы (`orders/new`, `pos`, `quotes`)
- Для позиции-услуги НЕ пред-заполнять редактируемую цену `basePrice`.
- При изменении услуги/кол-ва/размера/опций вызывать `POST /pricing/preview`
  `{ serviceId, quantity, sizeId?, optionIds?, area?, needsDesign? }` и подставлять
  `unitPrice`/`designSurcharge`/`lineTotal` из ответа.
- Ручная правка цены = «переопределение» (по желанию — под правом или с пометкой в заказе).

### (Опц.) Закрепление в `orders.create`/`quickSale`
- Для service-позиций пересчитывать цену на сервере (тот же `PricingService.calc`) и
  либо использовать её, либо запрещать цену ниже расчётной без права `pos.discountUnlimited`.
- Это защищает от занижения цены через API, но требует, чтобы формы слали `sizeId/optionIds`.
